# POI Marker Kit Project Plan

## 1. Objective 
Build a location-based-webxr kit that create beautiful markers given a real-world position:
- markers need to stay hidden when user is far away
- plays a striking reveal animation when the user gets close enough
- idles with subtle motion while in view
- grows into a focused state when the user taps or looks straight at it 
- plays a hide animation when the user walks away 
- markers hover well above their anchor, mark areas instead of exact points


## 2. High-level Architecture
### 2.1 Application type: 
- Web page with a small fixed three.js scene
- Another scene, this time using location-based-webxr

### 2.2 Module structrue
- `lifecycle.ts`
    - Stores marker states
    - Responsible for choosing a marker state, given the data
    - Responsible for switching through states
- `index.html`
    - Click through gallery
    - Fixed three.js test scene
    - Controllable camera
    - Loads markers 
- `marker_stub.ts`
    - Encapsulates marker information (name, icon, distance)
    - Baseline for all markers, each one is expanded from this given stub
- `attention.ts`
    - Responsible for detecting when an object is in view
- `main.ts`
    - WebXR integration 
    - Location based WebXR scene
- `distanceCalculation.ts`
    - Computes the distance between the user and the marker

## 3. State Semantics
 - `hidden`
    - Focus is away from the object
    - The user is a considerable distance away
- `revealing`
    - User gets close to the object
- `idle`
    - User is close to the object
    - The object is not in focus
- `focused`
    - User is close to the object
    - The object is in focus
- `hiding`
    - Focus is shifter away from the object
    - The user gets away from the object

## 4. Redux state setup
### 4.1 Actions
- `COMPUTE_FOCUS`
    - Payload: `{boolean: inFocus}`
- `COMPUTE_DISTANCE`
    - Payload: `{int: distance}`
- `CHANGE_STATE`
    -Payload: `{MarkerState: newState}`

### 4.2 Reducer behaviour
- `COMPUTE_FOCUS`
    - Based on the current user's perspective, calculates if the object is in view.
- `COMPUTE_DISTANCE`
    - Based on the current position of the user and the marker, calculates the distance between the two.
- `CHANGE_STATE`
    - Trigger a transition between two states.

## 5. 3D Scene and camera designs 
### 5.1 Scene setup
- Create a Three.js `Scene` with a `PerspectiveCamera`.
- Add ambient + directional lighting for marker visibility.
- Create a WebXR `Scene`.
- Add `THREE.CanvasTexture` and render onto a plane for UI  
### 5.2 Camera controls
- Use Three.js `OrbitControls` from `three/examples/jsm/controls/OrbitControls.js`.
### 5.3 Rendering markers
- For each marker in the scene, create a THREE.js `Object3D` object.
- Use a fixed marker style for each state
